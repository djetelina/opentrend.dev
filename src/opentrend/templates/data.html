Everything OpenTrend fetches, tracks, and computes for each project. All data is collected daily. Your data is not for sale.

{# ── GitHub ── #}

 GitHub

Via GitHub REST API and web scraping. Requires the project's GitHub repository.

Repository metrics

  • Stars, forks, watchers
  • Open & closed issues
  • Open & merged pull requests
  • Total contributors
  • Total commits
  • License (SPDX)
  • Release count, latest tag & date

Weekly statistics

  • Commit activity (per week)
  • Code frequency (additions & deletions)
  • Participation (owner vs community)

Dependents

  • Dependent repositories count
  • Dependent packages count

Scraped from dependency graph. Includes end-user apps, not just libraries.

Release downloads

  • Per-asset download counts per release
  • Artifact type detection (.deb, .rpm, .apk, .msi, etc.)

Traffic

  • Clones & unique clones per day
  • Page views & unique viewers per day
  • Top referrers (source, views, uniques)

GitHub retains traffic for 14 days only.

{# ── Reach Score ── #}

 Reach Score

Composite metric that summarizes a project's overall visibility. Computed daily.

{# Formula #}
Formula
reach =
  sqrt(stars)        × 30
+ sqrt(forks)        × 20
+ ln(1+contributors) × 40
+ ln(1+watchers)     × 20
+ ln(1+sources)      × 50
+ ln(1+downloads)    × 20
+ ln(1+dependents)   × 40
+ ln(1+views)        × 15
+ ln(1+clones)       × 15
{# Breakdown table #}
Input Scale Weight Source
Stars √ 30 GitHub API — cumulative, current total
Forks √ 20 GitHub API — cumulative, current total
Contributors log 40 GitHub API — cumulative, all-time
Watchers log 20 GitHub API — cumulative, current total
Package sources log 50 Distro & registry collectors — count of sources with a published version right now
Total downloads log 20 PyPI, npm, Crates.io, RubyGems, Packagist, NuGet, Homebrew, Chocolatey — monthly estimate (daily × 30 or last-month total)
Total dependents log 40 GitHub dependency graph — current total (fallback: npm, Crates.io, RubyGems, AUR)
Traffic views log 15 GitHub Traffic API — sum of last 30 days
Traffic clones log 15 GitHub Traffic API — sum of last 30 days

Stars and forks use square root scaling (diminishing returns at high counts). All others use log scaling. Zero inputs contribute nothing. Traffic and downloads use rolling windows, so the score can decrease when high-activity days roll off.

{# ── Other Computed Metrics ── #}

 Other Computed Metrics

Derived from collected data, not fetched from external sources.

Traction

  • Stars per week & per month
  • Issue close rate (7-day window)
  • PR merge rate (7-day window)

Packaging

  • Version freshness per source
  • Package breadth (% of ecosystems)
  • Total downloads across registries
  • Total dependents across sources

Deltas

Week-over-week change for stars, forks, issues, PRs, contributors, commits, watchers, and releases.

{# ── Collection Details ── #}

 Collection

How and when data is collected.

Frequency — once per day per project
Scheduling — staggered across 24h by project ID hash
Storage — append-only snapshots keyed by (entity, date)
Toggles — GitHub, traffic, and packages per project
